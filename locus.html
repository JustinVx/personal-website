Locus Map Pro aims to help outdoor enthusiasts reach their destination. However, Locus Map offers much more than just navigation: it facilitates users in every aspect of outdoor activities. Among others, these functions include choosing and uploading maps, recording tracks, saving tracks and points, sharing routes and geocaching. With all these options and functionalities comes a problem. Through an extensive user test we discovered that for most users it is hard to find out everything that is possible or to even use the functions that they need. This is what we aimed to change.

The design process

As with any redesign process, our process started with a thorough analysis of the current design. After that an extensive user test was done. This led to a number of conclusions that were used to make a design brief. This design brief was the start of a iterative redesign phase. The phase started with a creative session with many ideas on how to improve the design together and lead to an epiphany about the ultimate design goal.

Through a number of smaller iterations the design was continually improved. We used both paper prototypes and digital prototypes to see the changes in usability and saw improvements every time. Especially the smaller user tests really helped in developing the concept to something significantly more user friendly. An interactive prototype was made of the redesign an then extensively tested in a user test similar to our first user test. All the conclusions from this test were used for a final redesign that was then presented in a movie.

The first redesign

One of the main problems determined concerns the hierarchy and structure of the application. Therefore the first step of the redesigning process was to restructure the functions. When the application is opened the user should be guided through all the steps to create a route, without forcing the user. The new user interface design matches the users expectations that were found in the earlier user study with the original application.

Instead of only looking at what happens when using the app and what goes wrong, I proposed that we should also look into why people use the app. The main reason people use this app and not for instance Google Maps, is that they want to hike, bike or sail a nice route (and not necessarily the fastest route). During our redesign this was something that we focused on and really helped our design.

The whole outdoor experience and interaction with the application consist of three phases: preparation, journey and post journey. In the preparation phase the user creates a route. During the journey the user will have access to all the tools he/she would need on the road, like record track, current location, nearby points, camera, search and navigate. After the journey the user has the ability to evaluate and share the route and data.

In the preparation phase the user wants to set a route. To set a route four things are needed: the activity, a map that meets the users need of information, points the user wants to pass by and finally a route between all these points. Because these four steps are always necessary for creating a route the new interface will guide the user through these four steps. This makes it easier for beginning hikers to create a route and expert hikers will have a good overview. These steps should be guiding and not forcing. The user needs to have the possibility to go back and have an overview.

The second redesign

The steps for creating a route are designed to help users to plan a trip while they can alter any step at any given moment before and during their journey. In the newly designed menu the step sequence is again indicated but less highlighted through a less ‘forcing’ horizontally alignment. Steps do not necessarily need to be taken, as a default configuration is set depending on the current position/the last activity. This leads to a more suggestive approach as opposed to the strictly guiding approach of the previous redesign. The most important functions can now all be reached through just one step.
